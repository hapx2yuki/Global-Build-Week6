# 安全性・個人情報・費用の確認

## 秘密情報

- [ ] API キーは環境変数から読み、コード、README、ログ、画像、動画に含めない。
- [ ] ブラウザへ配布されるコードにサーバー用の秘密情報を埋め込まない。
- [ ] 開発用と提出用で認証情報を分ける。
- [ ] 漏えい時の失効と再発行の手順がある。

## 個人情報と保持

- [ ] 必要最小限の情報だけを収集する。
- [ ] 保存目的、保存期間、削除方法を説明できる。
- [ ] 試験と動画では合成データを使う。
- [ ] ログから氏名、メール、入力本文、認証情報を除く。

## AI 固有の危険

- [ ] 利用者の入力を命令とデータに分離する。
- [ ] GPT-5.6 の出力を構造と許可範囲で検証する。
- [ ] 外部操作の前に権限確認または人間の確認を置く。
- [ ] 誤答、根拠不足、拒否、時間切れの表示を設計する。
- [ ] 危険な用途、権限逸脱、指示の乗っ取りを試験する。

## 費用と可用性

- [ ] 一回当たりの呼び出し回数と最大入力長を制限する。
- [ ] 再試行回数、時間切れ、同時実行数を制限する。
- [ ] 無料の審査経路に利用上限と分かりやすい失敗表示がある。
- [ ] 外部サービス停止時に状態と再試行方法を示す。

## 提出前

- [ ] `make secret-check` が成功する。
- [ ] 公開 URL とリポジトリに本番情報がない。
- [ ] 動画の通知、ブラウザ履歴、個人アカウント表示を確認する。
- [ ] 依存物と素材の利用条件を `docs/third-party-assets.md` で確認する。
