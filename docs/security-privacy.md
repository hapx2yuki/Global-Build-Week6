# 安全性・情報管理

## 採用した境界

- 実データは端末内だけに保存する。公開体験版は架空データだけを使う。
- OpenAI APIキーを要求しない。`codex login status`と`codex login`だけを利用し、`~/.codex/auth.json`を読まない。
- GPT‑5.6への送信は承認済みの区分だけに限定し、実行ごとに承認を取り直す。
- 根拠内の命令文は引用データとして囲み、指示として扱わない。
- AI出力は`additionalProperties: false`のSchema、識別子、依存関係、引用位置、本文ハッシュを検査する。
- 高影響な人の評価、採用、融資、保険、医療、法律、教育合否は停止する。

## localhost防御

- ランダムな`127.0.0.1`ポートだけで待ち受ける。
- 起動ごとに256ビットの初期交換用・セッション用・CSRF用トークンを生成する。
- 初期交換用トークンはURL断片で一度だけ渡し、交換後にURLから消す。
- セッションはHttpOnly、SameSite=Strict Cookieで保持する。
- `Host`、`Origin`、`Sec-Fetch-Site`を検査し、CORSを許可しない。
- 状態変更にはCSRFヘッダーを要求する。
- CSP、`frame-ancestors 'none'`、`object-src 'none'`、権限方針を設定する。

## 原本処理

- MIME型とファイル署名を照合する。
- Office文書はZIP展開件数、展開容量、圧縮率、相対パスを検査する。
- XML外部実体、マクロ、埋込実行物、外部リンクを実行しない。
- 数式を実行せず、式と保存済み表示値を分ける。
- SVGは生HTMLで描画せず、`sharp`で安全な画像へ変換する。
- Gitでは`.gitignore`を尊重し、秘密情報、依存物、生成物、許可領域外シンボリックリンクを除外する。

## Codex修正

- 元HEADとdirty状態を固定し、別の一時worktreeを作る。
- 憲法はworktree外の読取専用ファイルとして置き、実行前後にSHA‑256を検査する。
- 変更ファイルを完全な許可一覧と禁止接頭辞で照合する。
- 変更後のシンボリックリンクがworktree外を指す場合は拒否する。
- 受け入れコマンドは引数配列、`shell: false`で実行する。
- 試験、憲法ハッシュ、変更範囲、Codex報告と実差分が全て一致した場合だけ人間へ適用確認を出す。
- 適用直前に元HEADとclean状態を再確認し、`git apply --check`後に適用する。

## 検査済み事項

- 指示乗っ取り文を含む根拠
- 未宣言AI出力項目
- 偽の引用ハッシュ
- ZIPパストラバーサルと高圧縮率
- 生SVG内のスクリプト
- 許可領域外Gitシンボリックリンク
- 許可外修正
- 元HEAD変更
- 私的引用の共有一式への混入
- 端末絶対パス、APIキー形式、トークン名の共有一式への混入
- 公開配備用成果物へのローカルAPI混入

## 残る危険

- 解析ライブラリ、Codex CLI、OSの未知の欠陥。
- 端末自体が侵害された場合。
- 利用者が送信確認を読まず、過剰な区分を承認する場合。
- 利用者が修正許可範囲を広く設定する場合。
- GPT‑5.6の将来更新による判定変化。異なる版は同一条件として比較しない。
- SSD上の削除済みブロックの物理完全消去。

公開前条件は、高・重大な依存物脆弱性0件、秘密情報検査成功、公開版分離検査成功、主要安全性試験成功とする。
