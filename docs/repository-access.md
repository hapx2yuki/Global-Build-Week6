# 審査用リポジトリ権限

## 現在の方針

開発中は非公開リポジトリを使う。提出前に、次の両方へ閲覧権限を付与する。

- `testing@devpost.com`
- `build-week-event@openai.com`

招待後は、保留中ではなく利用可能になっていることを確認し、`submission/metadata.env` の `REPOSITORY_ACCESS_CONFIRMED` を `true` にする。

## 確認手順

1. ログアウト状態または権限を持たない別アカウントで、非公開であることを確認する。
2. GitHub の共同作業者設定で、指定した審査用アカウントの権限を確認する。
3. README の導入手順を新しい作業ディレクトリで実行する。
4. 審査終了までリポジトリを削除、改名、非共有化しない予定を確認する。
5. Devpost の Repository URL が既定ブランチを正しく開くことを確認する。

## 公開へ変更する場合

外部コード、素材、データの権利を確認し、意図に合うライセンスを選んでリポジトリに明記する。ライセンスを選ぶ前に公開しない。
