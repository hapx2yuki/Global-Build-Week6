## 審査観点と証拠

| 審査観点 | 必ず用意する証拠 | 動画で見せる内容 | README で示す内容 |
| --- | --- | --- | --- |
| 技術実装 | 実機結果、試験、失敗時処理、GPT-5.6 の具体的役割 | 最難関の処理が動く瞬間 | 構成、導入、試験、制約 |
| 設計 | 初回利用の迷い、待ち時間、誤操作への対応 | 説明なしで完了できる主要経路 | 対象利用者と利用手順 |
| 潜在的な影響 | 基準値との比較、時間・品質・到達可能性の変化 | 導入前後の差 | 測定方法、対象範囲、限界 |
| アイデアの質 | 問題の実在性、既存手段との差、必然性 | 冒頭15秒の問題設定 | なぜ今、なぜ GPT-5.6 か |

## 数値の記録規則

- 比較対象、標本数、実行環境、測定日を併記する。
- 平均だけでなく、ばらつきと失敗数を記録する。
- 推定値を実測値のように書かない。
- デモ用の選別結果だけを全体性能として主張しない。
- 再現コマンドまたは画面記録への参照を付ける。
