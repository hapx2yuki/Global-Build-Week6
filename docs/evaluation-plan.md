# 評価計画と実測記録

## 中核仮説

| 項目 | 内容 |
| --- | --- |
| 対象利用者 | Codexで製品を作るが、コードを詳細には評価しない製品責任者 |
| 問題 | 実装速度に意図の明文化と検査が追いつかず、AIの仮定が本人の決定に混ざる |
| 代替手段 | 会話履歴、長い依頼文、都度の目視確認、再実行ごとに変わる採点 |
| 製品による変化 | 判断基準を人間承認済みの不可逆な憲法へ変え、同一条件で評価・修正・再評価する |
| 成功指標 | 5分以内の有用案、20分以内の作成可能性、重要結論の根拠率100%、必須判定3回一致 |
| 許容できない失敗 | 私的本文の共有、根拠の捏造、憲法の無断変更、揺れの多数決による隠蔽 |

## ローカル実経路の自己検証

2026-07-21に、公開再生データではなく、端末内保存、既存の`codex login`、`gpt-5.6-terra`を使う画面経路を実行した。

| 段階 | 実測 |
| --- | --- |
| 取込 | 架空Markdownを原文、行位置、本文ハッシュ付きで保存 |
| 憲法案 | 約37秒で8区分、3基準、重要質問2件を構造化生成 |
| 人間判断 | 表示された規則・例を承認し、重要質問2件へ回答 |
| 校正 | 同一設定3回が`stable / met`へ集約 |
| 作成 | 不可逆なv1.0を作成。内容ハッシュ`a02e559986895c6af6967c413f4e3be9cacfbb159ddfc76aeddcd2cb71bdc55b` |
| 成果物固定 | CriteriaForge自身のGitを314区分、184ファイル、コミットとdirty状態付きで固定 |
| 正式評価 | 3回を約50秒で完了し、`unstable / blocked`へ集約 |
| 復旧 | アプリを完全終了・再起動し、取込、下書き、判断、憲法版、対象、実行記録を復元 |

正式評価では3基準の根拠充足などが実行間で一致しなかった。これは成功扱いにせず、理由を項目ごとに表示し、修正依頼を無効にした。自己実証として重要なのは、都合のよい多数決で`met`や`not_met`へ丸めなかったことである。

## 自動試験の範囲

- JSON Schemaと未宣言項目拒否
- 8区分、5条件、四層集約
- 3回の設定同一性と不一致停止
- 意味ハッシュと依存先だけの無効化
- 引用の出所・位置・本文ハッシュ再検証
- SQLiteの復元、権限、削除、不可逆版更新拒否
- TXT、PDF、DOCX、PPTX、CSV、XLSX、画像、SVG、動画保留、Git
- ZIPパストラバーサル、高圧縮率、秘密情報除外、外向きシンボリックリンク
- Codex OAuth状態と構造修復1回
- 共有一式への私的引用・秘密・絶対パス混入拒否
- Git worktreeの許可内変更、許可外変更、追加ファイル、元HEAD変更
- 1440×900、390×844、キーボード、英日切替
- `axe`のcritical／serious違反0件とブラウザconsole error 0件
- `https://criteriaforge.vercel.app`に対する新規デスクトップ／スマートフォン文脈の4画面試験
