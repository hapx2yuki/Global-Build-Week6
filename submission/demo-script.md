# Demo Video Script

Target runtime: **160–170 seconds**. Hard limit: fewer than 180 seconds. The final video must have English audio, accurate captions, and public YouTube visibility.

## 0:00–0:15 — Problem and promise

- Show the real context before showing a title card.
- Say who has the problem, what is painful today, and what changes after using the product.
- Use one sentence that a non-specialist can repeat.

## 0:15–0:35 — Starting state

- Show the input or situation before the product acts.
- Explain the current workaround and why it fails.
- Avoid feature lists.

## 0:35–1:35 — Live primary journey

- Complete one end-to-end task with realistic sample data.
- Keep the cursor and screen readable.
- Show waiting, correction, and confirmation states rather than editing them out.
- Point out the moment where GPT-5.6 performs the difficult part.

## 1:35–2:05 — Technical proof

- Show the smallest useful architecture view.
- Explain GPT-5.6 input, output, validation, and safety boundary.
- Show a test, result record, or failure recovery.
- Explain one concrete way Codex accelerated development.

## 2:05–2:30 — Impact and comparison

- Compare the product with the current workaround using a verified example.
- State measurement conditions and limitations.
- Show why the difference matters to the target user.

## 2:30–2:45 — Close

- Restate the user outcome.
- Show the project name and repository or live-demo location.
- End before 2:50 to preserve export and platform margin.

## Recording checks

- [ ] No secret, personal notification, account email, browser history, or unrelated tab is visible.
- [ ] Text remains readable at 1080p.
- [ ] The first 15 seconds work without prior context.
- [ ] Audio is understandable without music.
- [ ] Captions match the spoken words.
- [ ] The exported file is shorter than 180 seconds.
- [ ] Public playback works while logged out and on another device.
- [ ] The URL pasted into Devpost opens the intended video.
